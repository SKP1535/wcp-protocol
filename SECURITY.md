# Security Policy

## Reporting Security Issues

If you discover a security vulnerability related to the Work Coordination Protocol reference implementation or tooling, please report it responsibly.

Do **not** publicly disclose the vulnerability immediately.

Instead:

1. Open a private issue or contact the maintainers
2. Provide detailed information about the vulnerability
3. Allow time for investigation and remediation

---

## Scope

This security policy applies to:

* reference implementations
* SDKs
* CLI tools
* protocol tooling

The protocol specification itself may evolve to address security considerations.

---

## Best Practices

Implementations of WCP should follow security best practices including:

* secure authentication
* proper authorization checks
* validation of work objects
* encrypted communication channels
* audit logging of work execution

---

## Future Security Enhancements

Future protocol versions may include:

* cryptographic event signing
* decentralized identity integration
* agent reputation systems
* secure distributed work coordination
